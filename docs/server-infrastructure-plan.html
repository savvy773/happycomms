핵심 원칙

4인 팀에서는 “직접 서버를 잘 돌리는 것”보다 관리형(managed) 서비스 + 자동화 백업 + 최소 권한이 생존에 유리합니다. Plan A는 이 원칙을 최우선으로 한 추천안이고, Plan B는 데이터 주권·월 고정비 최소화를 원할 때의 대안입니다.

03 추천 조합 — Plan A / Plan B

4인 팀 기준 기술 스택 제안

추천
Plan A · Cloud First

관리형 클라우드 올인원

운영 인력 없이 안정적으로 굴리기 — 1순위 추천

  • 호스팅 AWS ap-northeast-2 (서울) 또는 Naver Cloud Platform
    소형 VM 1~2대 + 관리형 DB 분리
  • 컴퓨트 Docker Compose on EC2 t3.small / NCP Micro
    또는 AWS Lightsail / NCP Server 고정 요금 패키지
  • DB PostgreSQL 16 관리형
    (RDS / NCP Cloud DB) — 자동 백업·암호화
  • 파일/백업 S3 / Object Storage — 계약서·증빙·리뷰 이미지
    일 단위 DB 스냅샷 + 주 단위 오프사이트 백업
  • 앱 구조 관리자·인트라넷: Next.js (또는 NestJS + React)
    API: REST/JSON · 인증: Auth.js / Keycloak lite
  • 인트라넷 사내 전용 웹 (VPN 또는 SSO + IP/디바이스 제한)
    문서: 자체 위키 모듈 또는 Notion 연동(비민감만)
  • 캐시·큐 초기: 생략 가능 / 성장 시 Redis (Upstash 또는 동일 VPC)
  • 모니터링 Uptime Kuma + CloudWatch/NCP Monitoring
    알림: 슬랙/디스코드 웹훅
  • 도메인·TLS Cloudflare DNS + 무료 TLS · admin. / intranet. 서브도메인 분리

장점

  • 백업·패치·장애 대응 부담 낮음
  • 서울 리전으로 지연·법 이슈 완화
  • 트래픽 늘어나면 스케일 용이
  • 개인정보 암호화·스냅샷 기본 지원

단점

  • 월 비용이 고정+사용량
  • 클라우드 계정 권한 관리 필요
  • 벤더 종속 일부 존재
예상 월 비용 (4인 · 초기)
₩8만 ~ ₩25만

Plan B · Mini Server Hybrid

사무실 미니서버 + 클라우드 백업

데이터 주권·월 고정비 최소화 — 대안 추천

  • 하드웨어 미니 PC / NUC급 1대 (RAM 32GB, SSD 1TB+, UPS 필수)
    또는 저가 NAS(TrueNAS / Synology) + 앱 서버 분리
  • 하이퍼바이저 Proxmox VE — VM/LXC로 DB·앱·인트라넷 분리
    스냅샷 롤백으로 실수 복구
  • 런타임 Docker Compose 스택 동일 구성
    PostgreSQL + App + Reverse Proxy (Caddy / Traefik)
  • DB 자체 PostgreSQL (볼륨 암호화 + 일 백업)
    민감 스키마 분리 (hr / finance 스키마·별도 DB 권장)
  • 외부 노출 Cloudflare Tunnel 또는 Tailscale Funnel
    공인 IP 직접 개방 금지 · 방화벽 최소 포트
  • 원격 접속 Tailscale / WireGuard — 직원 VPN
    인트라넷은 VPN 내부에서만 접근
  • 오프사이트 백업 암호화 후 S3 / Backblaze B2 / NCP Object 일 1회
    사무실 화재·도난 대비 필수
  • 인트라넷 동일 앱 스택 또는 BookStack / Outline (셀프호스트 위키)
    파일: Nextcloud (선택)
  • UPS·물리 UPS, 케이블 정리, 디스크 이중화(가능 시), 잠금 캐비닛

장점

  • 월 운영비 낮음 (전기+백업)
  • 데이터 물리 위치 통제
  • 실험·내부 도구 자유도 높음

단점

  • 정전·HW 고장 시 본인이 대응
  • 보안 패치·모니터링 직접
  • 초기 장비비 + 학습 비용
  • 외근 시 접속 품질 의존
예상 초기 + 월 비용
초기 ₩80만~150만 · 월 ₩2만~8만
미니 PC+UPS+SSD 1회 투자, 월은 전기·도메인·오프사이트 백업 위주.

06 보안 · 개인정보 최소 기준

1
전송·저장 암호화 전 구간 TLS 1.2+ · DB at-rest 암호화 · 백업 파일 AES 암호화

6
개인정보 처리방침·동의 수집 목적·보관 기간·파기 절차 문서화 · 퇴사 시 계정 즉시 회수

네트워크 분리 DB 포트 외부 비공개 · 인트라넷은 VPN 또는 IP allowlist
8

⚖️
법적 참고 (요약)

고객·인플루언서 연락처·주소 등은 개인정보에 해당합니다. 소규모여도 수집 최소화, 목적 외 이용 금지, 안전 조치(접근통제·암호화·로그)는 기본입니다. 상세 의무는 규모·처리 형태에 따라 달라지므로 실제 운영 전에는 개인정보 보호 담당 또는 전문 자문을 권장합니다. (본 문서는 법률 자문이 아닙니다.)

08 단계별 로드맵

한 번에 다 짓지 말고 위험 순으로

Phase 0 · 1주

계정·도메인·보안 기초

클라우드(또는 미니 PC) 확보, 도메인/DNS, Cloudflare, 관리자 MFA, 비밀번호 매니저(1Password/Bitwarden) 도입. 데이터 분류표 확정.

Phase 1 · 2~3주

핵심 DB + 관리자 CRM MVP

PostgreSQL 기동, 고객·거래처·인플루언서·캠페인 CRUD, 역할 기반 로그인, 일 백업 자동화. 스프레드시트 데이터 1차 이관.

Phase 2 · 2주

인트라넷 · 문서 · 직원 모듈

공지/위키, 직원 마스터, 권한 매트릭스. 연봉·매출 화면은 별도 메뉴 + 감사 로그 필수. VPN 또는 관리자 IP 제한.

Phase 3 · 지속

정산 고도화 · 외부 포털 · 모니터링

체험단 정산 워크플로, 인플루언서 셀프 조회 포털(선택), 업타임·디스크 알림, 분기 복구 리허설, 개인정보 파기 스케줄.

09 최종 권고

지금 당장 고른다면 → Plan A

DevOps 전담이 없고, 고객·인플루언서 개인정보와 매출 데이터를 안정적으로 지켜야 하는 4인 팀에는 서울 리전 관리형 PostgreSQL + Docker 앱 1대 + S3 + Cloudflare 조합이 가성비·리스크 균형이 가장 좋습니다.

  • 추천 패키지 예: AWS Lightsail 또는 NCP Server + Cloud DB for PostgreSQL
  • 앱: Next.js monorepo (admin + intranet)
  • 접속: admin / intranet 서브도메인 분리, MFA 필수

Plan B를 고를 조건

팀 내 인프라에 익숙한 인원이 있고, 월 클라우드 비용을 극단적으로 줄이며 데이터가 사무실에 있기를 원할 때. 단, 오프사이트 암호화 백업 + Tailscale + UPS는 협상 불가 조건입니다.

  • Proxmox + Docker Compose 동일 스택
  • Cloudflare Tunnel로 공인 포트 최소화
  • 장애 시 대표/담당 응답 체계 문서화
